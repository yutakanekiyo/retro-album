## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| **プロジェクト名** | （仮）Retro Scrap — サークル先輩への贈り物アプリ |
| **コンセプト** | 自動生成スクラップブック × 無限スクロール × BGM |
| **対象ユーザー** | 卒業/引退する先輩 11人以上 |
| **開発者** | Mac使用、HTML/CSS/JS経験あり、バイブコーディング経験豊富 |

| **将来計画** | PWA公開後、iOS/Androidネイティブアプリとしてリリース |

## 2. コアコンセプト

### 「紙のスクラップブックの温かみ × デジタルの自動生成・ランダム性」

先輩がログインすると、自分の写真50枚以上が**スクラップブック風のコラージュ**として自動レイアウトされている。BGMが流れる中、無限スクロールで次々とページが現れる。

**紙のアルバムとの決定的な違い:**
- 開くたびにレイアウトが完全ランダムに変わる → 何度見ても新鮮
- 50枚以上の写真が自動でおしゃれに配置される → 制作コスト不要
- BGMと一緒に体験する → 没入感
- デジタルだからこそのアニメーション・インタラクション

**体験設計のポイント:**
- 「その場で盛り上がる」が最重要 → 追いコン等で先輩に渡し、その場でワイワイ見る
- 先輩は自分のスマホで個別に見る
- 後からも見返せるが、「毎日開く理由」系の機能は不要
- 贈呈の瞬間がピーク → その瞬間のインパクト最大化

## 3. 技術スタック

### フロントエンド
- **Next.js**（React ベース、PWA化が容易）
- **Tailwind CSS**（高速なスタイリング）
- **Framer Motion**（アニメーション・トランジション）
- **next-pwa** または **Serwist**（PWA化）

### バックエンド / インフラ
- **Supabase**（課金済み）
  - **Auth**: 先輩ごとのログイン認証
  - **Storage**: 写真・動画・BGMの保存
  - **Database (PostgreSQL)**: ユーザー情報、アルバムデータ管理
  - **Row Level Security (RLS)**: 各先輩が自分のアルバムだけ見れるようにする

### デプロイ
- **Vercel**（無料枠）

### 開発ツール
- **Claude Code**（メイン開発ツール）

## 4. 画面構成

アプリは大きく分けて **3つの画面** で構成する。

```
[ログイン画面] → [スクラップブック画面（メイン）] ↔ [ギャラリー画面]
```

---

### 4.1 ログイン画面

先輩がアプリに入る最初の体験。ここからすでに「おしゃれ」を感じてもらう。

| 要素 | 詳細 |
|------|------|
| ログイン方式 | メールアドレス＋パスワード（Supabase Auth） |
| アカウント | 管理者が事前に先輩分を作成済み |
| デザイン | レトロ・フィルム風。サークル名やロゴ入り |
| 演出 | ログインボタン押下後、スクラップブックが開くようなトランジション |
| セッション | 一度ログインしたら保持（再ログイン不要） |

---

### 4.2 スクラップブック画面（メイン画面）

**アプリの核心。** 写真が自動でコラージュ/スクラップブック風にレイアウトされる画面。

#### レイアウトエンジン

| 要素 | 詳細 |
|------|------|
| 構造 | 「ページ」単位のセクションが縦に並ぶ無限スクロール |
| 1ページあたり | 5〜8枚の写真をコラージュ配置 |
| ページ数 | 写真50枚 ÷ 5〜8枚 ≒ 約7〜10ページ |
| レイアウト方式 | 複数のテンプレートパターンをランダムに適用 |
| ランダム性 | 毎回完全ランダム（写真の組合せ・配置・角度・サイズ全て） |

#### テンプレートパターン（例）

レイアウトのバリエーションを複数用意し、ページごとにランダムに選ばれる:

- **パターンA**: 大きい写真1枚 + 小さい写真4枚が周囲に散らばる
- **パターンB**: 2×3のグリッドベースだが角度・サイズがランダムにずれる
- **パターンC**: 中央にポラロイド写真、周囲にステッカーとテキスト
- **パターンD**: 密度高めのコラージュ（盛岡という星で風）
- **パターンE**: 横に並ぶフィルムストリップ風
- **パターンF**: 斜めに重なり合うポラロイド風

#### デコレーション要素

写真に自動で付与されるデコレーション（ランダムに組み合わせる）:

| 要素 | 実装方法 |
|------|----------|
| **ポラロイド枠** | CSS box-shadow + 白い padding + 下部にキャプションスペース |
| **マスキングテープ** | SVG or CSS疑似要素、角度・色・位置をランダム化 |
| **ステッカー・スタンプ** | SVG素材を事前に10〜20個用意、ランダム配置 |
| **手書き風テキスト** | 手書きフォント（例: Caveat, Klee One）で日付やコメント |
| **切り抜き風文字** | 雑誌の切り貼り風タイポグラフィ（異なるフォント・サイズ・色・背景を1文字ずつ） |
| **フィルムグレイン・ノイズ** | CSS background-image でノイズテクスチャを全体にオーバーレイ |

#### 背景テクスチャ

ページごとにランダムで背景を変える:
- クラフト紙風（ベージュ系）
- グリッドノート風（方眼紙）
- コンクリート/壁風（グレー系）
- コルクボード風
- ダークトーン（黒板風）

#### インタラクション

| 操作 | 動作 |
|------|------|
| スクロール | 次のページが現れる（無限スクロール） |
| 写真タップ | フルスクリーン表示（拡大して見る） |
| スクロールアニメーション | ページがスクロールで視界に入ると、写真が順番にフェードイン or スライドイン |

#### BGM

| 要素 | 詳細 |
|------|------|
| 再生タイミング | ログイン後、最初のユーザー操作で再生開始（iOS Safari制約対応） |
| コントロール | 再生/一時停止、ミュートボタン（画面端に常時表示、控えめなデザイン） |
| ループ | 曲が終わったらリピート |
| 音源 | 著作権フリーBGMをSupabase Storageに格納 |

#### 動画対応

| 要素 | 詳細 |
|------|------|
| 配置 | 写真と同じようにスクラップブック内に配置 |
| 再生 | タップで再生（スクラップブック内はサムネイル or 短いプレビュー） |
| 形式 | MP4推奨 |

---

### 4.3 ギャラリー画面（Retro風の一覧画面）

スクラップブックとは別に、**写真が綺麗に整頓された一覧画面**。Retroアプリのような洗練されたグリッド表示。

| 要素 | 詳細 |
|------|------|
| レイアウト | 整然としたグリッド表示（Retro風） |
| 並び順 | 時系列 or 管理者が設定した順 |
| 写真タップ | フルスクリーン表示 → 左右スワイプで前後の写真へ |
| デザイン | ダーク背景、ミニマルでおしゃれ |
| 切り替え | スクラップブック画面 ↔ ギャラリー画面をタブ or ボタンで行き来 |

**スクラップブックが「体験」で、ギャラリーが「保存」。** ワイワイ見るときはスクラップブック、後から特定の写真を探すときはギャラリー。

### テーマ: エモい × レトロ × スクラップブック

| 要素 | 詳細 |
|------|------|
| **カラーパレット** | 暖色ベース（アンバー、セピア、クリーム、ダークブラウン） |

| **全体のノイズ** | フィルムグレインを薄くオーバーレイ（CSS） |
| **モード** | ダークベースをメイン（ギャラリー画面）、スクラップブックはページ背景による |
| **レスポンシブ** | モバイルファースト（iPhone Safariが最優先） |

## 6. データ設計（Supabase）

### テーブル構成

```
users（Supabase Auth で管理）
├── id (uuid)
├── email
└── password

profiles
├── id (uuid, FK → users.id)
├── display_name (先輩の名前)
├── avatar_url (プロフィール画像)
└── created_at

albums
├── id (uuid)
├── user_id (uuid, FK → users.id)
├── title (アルバムタイトル)
├── description (アルバムの説明)
├── bgm_url (BGMファイルのURL)
└── created_at

album_items
├── id (uuid)
├── album_id (uuid, FK → albums.id)
├── type ('photo' | 'video')
├── file_url (Supabase StorageのURL)
├── thumbnail_url (サムネイルURL)
├── caption (テキストキャプション)
├── date_label (表示用の日付ラベル: "2024.8.15" など)
├── sort_order (ギャラリー画面での表示順)
└── created_at
```

### Storage バケット構成

```
album-photos/
├── {user_id}/
│   ├── photo_001.jpg
│   └── ...

album-videos/
├── {user_id}/
│   ├── video_001.mp4
│   └── ...

bgm/
├── track_01.mp3
└── ...

decorations/
├── stickers/
│   ├── star.svg
│   ├── heart.svg
│   └── ...
├── tapes/
│   ├── tape_brown.svg
│   ├── tape_red.svg
│   └── ...
└── textures/
    ├── kraft_paper.jpg
    ├── grid_paper.jpg
    ├── concrete.jpg
    └── ...
```

### Row Level Security (RLS)

```sql
-- 各ユーザーは自分のアルバムだけ閲覧可能
CREATE POLICY "Users can view own album"
ON albums FOR SELECT
USING (auth.uid() = user_id);

CREATE POLICY "Users can view own album items"
ON album_items FOR SELECT
USING (
  album_id IN (
    SELECT id FROM albums WHERE user_id = auth.uid()
  )
);
```

## 8. 非機能要件

| 項目 | 要件 |
|------|------|
| パフォーマンス | 画像の遅延読み込み（Lazy Loading）、スクロール位置に応じた段階的描画 |
| 画像最適化 | アップロード時にWebP変換 & リサイズ（フル: 1200px幅、サムネ: 400px幅） |
| セキュリティ | RLSで他人のアルバムにアクセス不可、Storage URLは署名付き |
| 同時アクセス | 贈呈日に11人以上が同時アクセスしても問題なし |
| モバイル対応 | iPhone Safari を最優先で最適化 |
| オフライン | Service Workerで一度閲覧した写真をキャッシュ（後から見返せるように） |

## 13. 既知の問題（要対応）

### 写真が表示されないバグ
- Supabase StorageのURLが正しく生成されているか確認
- RLSポリシーがStorageにも適用されているか確認
- 署名付きURL（signed URL）の有効期限が切れていないか確認
- CORSの設定を確認
- Next.js の `next/image` を使う場合、`remotePatterns` にSupabaseドメインを追加
