# 📸 サークル贈呈用デジタルアルバムアプリ — 要件定義書

---

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| **プロジェクト名** | （仮）Retro Album — サークル先輩への贈り物アプリ |
| **目的** | サークルの先輩一人ひとりに、パーソナライズされたデジタルフォトアルバムを贈る |
| **対象ユーザー** | 卒業/引退する先輩 11人以上 |
| **開発者** | あなた（Mac使用、HTML/CSS/JS経験あり、バイブコーディング経験豊富） |
| **開発期間** | 1週間以内（PWA版） |
| **将来計画** | PWA公開後、iOS/Androidネイティブアプリとしてリリース |

---

## 2. 開発戦略：2段階ローンチ

### Phase 1: PWA版（1週間以内）← 今回のスコープ
- Webアプリとしてデプロイ、iPhoneのホーム画面に追加して使用
- 審査不要、即公開可能
- 贈呈日に間に合わせることが最優先

### Phase 2: ネイティブアプリ化（将来）
- React Native / Expo で PWA のコードを活かしつつネイティブ化
- App Store / Google Play に公開
- プッシュ通知やオフライン完全対応など追加機能

## 3. 技術スタック（推奨）

### フロントエンド
- **Next.js**（React ベース、PWA化が容易）
- **Tailwind CSS**（高速なスタイリング）
- **Framer Motion**（スワイプ・トランジションのアニメーション）
- **next-pwa** または **Serwist**（PWA化プラグイン）

### バックエンド / インフラ
- **Supabase**（課金済み）
  - **Auth**: 先輩ごとのログイン認証
  - **Storage**: 写真・動画の保存（50枚以上 × 11人 = 550枚以上）
  - **Database (PostgreSQL)**: ユーザー情報、アルバムデータ、メッセージ管理
  - **Row Level Security (RLS)**: 各先輩が自分のアルバムだけ見れるようにする

### デプロイ
- **Vercel**（Next.jsとの相性最高、無料枠で十分）

### 開発ツール
- **Claude Code**（メイン開発ツール）
- **VS Code**（補助エディタ）
- **Git / GitHub**（バージョン管理）

## 4. 機能要件

### 4.1 認証・ログイン

| 機能 | 詳細 |
|------|------|
| **ログイン方式** | メールアドレス＋パスワード（Supabase Auth） |
| **アカウント作成** | 管理者（あなた）が事前に先輩分のアカウントを作成 |
| **配布方法** | 先輩に「このURLにアクセスして、このアカウントでログインしてね」と伝える |
| **ログイン画面** | サークル名やロゴ入りのオリジナルデザイン |
| **セッション維持** | 一度ログインしたら保持（再ログイン不要） |

### 4.2 アルバムビューア（メイン機能）

| 機能 | 詳細 |
|------|------|
| **スワイプ操作** | 左右スワイプで写真を切り替え（Retro風のトランジション） |
| **写真表示** | フルスクリーン表示、ピンチズームに対応 |
| **写真枚数** | 1人あたり50枚以上を想定 |
| **並び順** | 時系列 or カテゴリ別（管理者が設定） |
| **サムネイル一覧** | グリッド表示で一覧を見渡せるビュー |
| **写真キャプション** | 各写真にテキスト（日付、コメント）を添付可能 |

### 4.3 動画再生

| 機能 | 詳細 |
|------|------|
| **動画挿入** | アルバムの写真と同じ流れの中に動画も配置 |
| **再生** | インライン再生（全画面切替も可） |
| **形式** | MP4推奨（ブラウザ互換性） |
| **注意点** | Supabase Storageの容量制限に注意（Proプランで100GB） |

### 4.4 BGM・音楽

| 機能 | 詳細 |
|------|------|
| **BGM再生** | アルバム閲覧中にバックグラウンドで音楽を再生 |
| **操作** | 再生/一時停止ボタン、ミュートボタン |
| **音源** | 著作権フリーのBGMをSupabase Storageに格納 |
| **自動再生** | iOS Safariの制約上、ユーザー操作後に再生開始 |
| **ループ** | 曲が終わったらリピート再生 |

### 4.5 テキストメッセージ（寄せ書き的な）

> ※ 今回ヒアリングでは未選択でしたが、贈り物として強く推奨する機能です

| 機能 | 詳細 |
|------|------|
| **表示** | アルバムの冒頭 or 末尾に寄せ書き風メッセージページ |
| **内容** | 管理者が事前に登録したテキスト |
| **将来拡張** | 他のサークルメンバーがメッセージを投稿できる機能 |

### 4.6 デザイン・UI/UX

| 要素 | 詳細 |
|------|------|
| **テーマ** | エモい・レトロ調（フィルム風） |
| **カラーパレット** | 暖色ベース（アンバー、セピア、クリーム、ダークブラウン） |
| **フィルム風エフェクト** | 写真にフィルムグレイン、ビネット、色褪せ加工 |
| **トランジション** | スライド切替時にフェードやフィルムスライド風のアニメーション |
| **フォント** | レトロ感のある手書き風or セリフ体（例: Noto Serif JP + 手書き風英語フォント） |
| **UI要素** | フィルムのパーフォレーション（穴）風のデコレーション |
| **ダークモード** | ダークベースが基本（フィルム/映写室をイメージ） |
| **レスポンシブ** | モバイルファースト（iPhoneでの表示が最優先） |

## 5. データ設計（Supabase）

### テーブル構成

```
users（Supabase Auth で管理）
├── id (uuid)
├── email
└── password

profiles
├── id (uuid, FK → users.id)
├── display_name (先輩の名前)
├── avatar_url (プロフィール画像)
└── created_at

albums
├── id (uuid)
├── user_id (uuid, FK → users.id) ← この先輩のアルバム
├── title (アルバムタイトル)
├── description (アルバムの説明)
├── bgm_url (BGMファイルのURL)
└── created_at

album_items
├── id (uuid)
├── album_id (uuid, FK → albums.id)
├── type ('photo' | 'video')
├── file_url (Supabase StorageのURL)
├── thumbnail_url (サムネイルURL)
├── caption (テキストキャプション)
├── sort_order (表示順)
└── created_at

messages
├── id (uuid)
├── album_id (uuid, FK → albums.id)
├── author_name (送り主の名前)
├── content (メッセージ本文)
├── sort_order (表示順)
└── created_at
```

### Storage バケット構成

```
album-photos/
├── {user_id}/
│   ├── photo_001.jpg
│   ├── photo_002.jpg
│   └── ...

album-videos/
├── {user_id}/
│   ├── video_001.mp4
│   └── ...

bgm/
├── track_01.mp3
└── ...

avatars/
├── {user_id}.jpg
└── ...
```

### Row Level Security (RLS) ポリシー

```sql
-- 各ユーザーは自分のアルバムだけ閲覧可能
CREATE POLICY "Users can view own album"
ON albums FOR SELECT
USING (auth.uid() = user_id);

-- アルバムアイテムも同様
CREATE POLICY "Users can view own album items"
ON album_items FOR SELECT
USING (
  album_id IN (
    SELECT id FROM albums WHERE user_id = auth.uid()
  )
);

-- Storageも同様に自分のフォルダだけアクセス可能
```

## 6. 管理者機能（あなた用）

| 機能 | 方法 |
|------|------|
| **アカウント作成** | Supabase Dashboard または CLI スクリプト |
| **写真アップロード** | 管理画面（簡易的なものをWebで作成）or Supabase Dashboard 直接操作 |
| **アルバム編集** | 管理画面から写真の順番変更、キャプション編集 |
| **一括アップロード** | CLIスクリプトでフォルダ単位でまとめてアップロード |

### 推奨: CLIアップロードスクリプト

先輩ごとのフォルダに写真を入れておき、一括でSupabaseにアップロードするNode.jsスクリプトを作成する。

```
upload-data/
├── tanaka_senpai/
│   ├── photo_001.jpg
│   ├── photo_002.jpg
│   └── video_001.mp4
├── suzuki_senpai/
│   ├── photo_001.jpg
│   └── ...
└── ...
```

## 7. 非機能要件

| 項目 | 要件 |
|------|------|
| **パフォーマンス** | 画像の遅延読み込み（Lazy Loading）、サムネイル生成で初期表示を高速化 |
| **画像最適化** | アップロード時にWebP変換 & リサイズ（フル: 1200px幅、サムネ: 300px幅） |
| **オフライン対応** | Service Workerで一度閲覧した写真をキャッシュ |
| **セキュリティ** | RLSで他の先輩のアルバムにアクセス不可、Storage URLは署名付き |
| **同時アクセス** | 贈呈日に11人以上が同時アクセスしても問題なし（Supabase Proで十分） |
| **モバイル対応** | iPhone Safari を最優先で最適化 |

## 9. 先輩への配布方法

1. **URL共有**: QRコード or リンクを先輩に送る
2. **ログイン情報**: 先輩一人ひとりにメール＋パスワードを渡す（紙のカードなどにするとオシャレ）
3. **ホーム画面追加の案内**: 「Safariで開いて、共有ボタン→ホーム画面に追加」の手順を図解で用意

### 配布カードのイメージ
```
┌──────────────────────────────┐
│  🎞️ あなただけのアルバム      │
│                              │
│  URL: album.example.com      │
│  [QRコード]                   │
│                              │
│  メール: tanaka@album.app    │
│  パスワード: xxxxxxxx        │
│                              │
│  ※ Safariで開いてね！        │
└──────────────────────────────┘
```

## 11. 将来のネイティブ化に向けて

PWAからネイティブアプリへ移行する際のポイント:

1. **React Native (Expo)** への移行が最もスムーズ
   - Next.js の React コンポーネントを多くの部分で再利用可能
   - Supabase との連携もそのまま使える
2. **ネイティブ化で追加できる機能**
   - プッシュ通知（新しい写真が追加された時など）
   - 完全なオフライン対応
   - より滑らかなアニメーション
   - 写真のダウンロード/保存機能
3. **App Store 公開の準備**
   - Apple Developer Program 登録（年間12,800円）
   - アプリ審査への対応
